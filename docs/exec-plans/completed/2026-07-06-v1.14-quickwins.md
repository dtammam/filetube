# v1.14.0 "Quick Wins"

## Goal

Ship four small, additive quality-of-life improvements to FileTube's browsing
experience — a random/"feeling lucky" sort with a re-roll button, a consistent
FileTube wordmark on mobile, a per-folder sidebar-hide flag distinct from the
existing "Hide from home" flag, and a configurable default landing view — with
zero behavioral change for anyone who doesn't touch the new options.

## Scope

1. **Random ("feeling lucky") sort + re-roll.** A new `random` option in the
   sort dropdown (`public/index.html:104-111`) that Fisher-Yates shuffles the
   current item list in `renderSorted()` (`public/js/main.js:104-116`); the
   `random` choice persists via `localStorage['filetube_sort']` exactly like
   the existing five sorts (`main.js:17` read / `:207` write). A small
   "shuffle again" re-roll button, visible only when `currentSort === 'random'`,
   re-randomizes the current item order in place without changing the
   selected sort option.
2. **Mobile logo = desktop wordmark.** On mobile viewports, the top-left brand
   mark becomes the same styled "FileTube" text wordmark used on desktop
   (`File<span class="tube">Tube</span>`, `public/index.html:54`,
   `public/css/style.css:284-301`) instead of the current `<img
   src="/favicon.svg">` icon (`.mobile-logo`, `index.html:55-57`, CSS
   `:1593`/`:1709-1722`). Applies to every page sharing this header:
   `public/index.html`, `public/setup.html`, `public/watch.html`. The v1.9.0
   mobile header layout (logo top-left, search full-width below) is otherwise
   unchanged.
3. **Hide a folder from the sidebar entirely.** A new, independent per-folder
   flag `folderSettings[path].hiddenFromSidebar`, distinct from the existing
   `folderSettings[path].hidden` ("Hide from home", server-side home/recent
   filter only, `server.js:1748-1756`). A new Setup toggle next to "Hide from
   home" (`public/setup.html:267`). `renderSidebarFolders`
   (`public/js/main.js:84-101`) omits a flagged folder from the sidebar list;
   the folder remains fully browsable via a direct `/?root=<path>` link and
   keeps its own "Hide from home" behavior independently.
4. **Default landing view.** A Setup dropdown to choose what renders on a bare
   home load: "Most Recent" (default) or any configured/mapped folder,
   identified by its existing folder path/key (the same string used as the
   `folderSettings` key, `item.rootFolder`, and the `?root=` param). Persisted
   as a new `db.settings.defaultView` preference, exposed via `GET`/`POST
   /api/settings`. Applied only on a bare home load (no `?search=`, `?folder=`,
   or `?root=` in the URL) in `loadLibrary()` (`public/js/main.js:33-81`); any
   explicit deep-link query param always wins over the default.

## Out of scope

- Server-side sort (sort stays client-side in `renderSorted()`; the server
  continues to always return newest-first, `server.js:1784`).
- The transcode pipeline, FFmpeg/FFprobe logic, and the watch page playback
  path (other than the shared header markup for item 2).
- The era-theme token set itself (no new CSS custom properties; new/changed
  CSS must consume existing `--var` tokens only).
- The `lib/ytdlp/` subscription module.
- Any new runtime dependency.
- **Cross-check against `docs/CONTRIBUTING.md`:** none of the above
  out-of-scope items defer anything CONTRIBUTING.md mandates for this change
  (tests, lint, XSS-safe dynamic strings) — those remain in scope for all four
  items below. No conflict found.

## Constraints

- **Additive / no regressions.** Existing sort options (newest/oldest/title-asc/
  title-desc/size-desc/size-asc), sidebar rendering, the home/recent view, the
  existing "Hide from home" toggle, and the v1.9.0 mobile header layout must be
  byte-identical in behavior for users who don't touch any of the four new
  options.
- **Code style:** vanilla DOM (no framework/bundler), CommonJS server,
  2-space/semicolons/single-quotes (`docs/CONTRIBUTING.md`). Prefer
  `textContent` over `innerHTML` for new dynamic strings; the existing sidebar
  render uses `escapeHtml()` + `innerHTML` — any new dynamic label added to
  that same render path must stay XSS-safe via the same `escapeHtml()`
  pattern (or `textContent`) — never raw interpolation.
- **Era-theme preserved.** Any new CSS must use existing `--var` tokens; no
  hardcoded colors that would break a theme.
- **Item 3 whitelist gotcha (hard constraint):** `POST /api/config` sanitizes
  `folderSettings` entries to `{name, hidden}` (+ conditional `order`)
  (`server.js:1532-1535`). `hiddenFromSidebar` MUST be added to that
  whitelist/sanitizer or a Setup save silently drops it. Backfill:
  `undefined` → `false` (not hidden).
- **Item 4 allowlist gotcha (hard constraint):** `POST /api/settings` rejects
  any key not in `KNOWN_KEYS` with `400` (`server.js:1643`). `defaultView`
  must be added end-to-end: `DEFAULT_SETTINGS` (`server.js:40-45`),
  `KNOWN_KEYS`, a validator (reject non-string / non-empty-sentinel values
  cleanly — never 500), and `settingsResponse`.
- **Deep links always win.** Item 4's default applies ONLY on a bare home load
  (no `?search=`/`?folder=`/`?root=`). A stored `defaultView` folder that no
  longer exists in `db.folders` falls back to Most Recent without error.
- **No new dependencies.**
- Tests: `node:test`, following the existing `test/unit/` /
  `test/integration/` split. Lint must pass with zero warnings.

- [ ] **[UNIT]** All five existing sort cases (`oldest`, `title-asc`,

### Item 4 — Default landing view

- [ ] **[UNIT]** `POST /api/settings` accepts a `defaultView` key without
      400ing (regression test against the `KNOWN_KEYS` allowlist gotcha at

- [ ] **[UNIT]** `DEFAULT_SETTINGS` includes a `defaultView` default (sentinel

- [ ] **[INTEGRATION]** `GET /api/settings` after a `POST` round-trips a

- [ ] **[INTEGRATION or MANUAL]** On a bare home load (`/` with no query

- [ ] **[UNIT or MANUAL]** A bare home load with `defaultView` set to a
      folder path that no longer exists in `db.folders` falls back to Most

- [ ] **[UNIT or MANUAL]** An explicit deep link (`/?root=...`, `/?search=...`,

- [ ] **[MANUAL — Dean on-device]** The Setup dropdown lists "Most Recent"

- [ ] **[MANUAL]** A folder flagged `hiddenFromSidebar` (item 3) can still be
