# v1.37.0 — Books (library + reader + progress), with v1.37.1 TTS seams

Status: Design APPROVED by owner intent (13 locked answers, 2026-07-13); implementation in progress on `release/v1.37.0`.
Owner vision: "FileTube isn't YouTube with extra steps. It's a centralized media platform for audio, video, text."

## Locked requirements (owner)

- Formats: EPUB + PDF only. Arrival: mounted folder(s), scanned. Scale: hundreds.
- Books page with cover-art cards (era styling), books in global search, continue-reading row on home.
- Reader: font size, themes, TOC, remembered position; desktop + mobile PWA first-class.
- Progress: position + percent. Shelves integrate with the existing pinned-playlists system.
- TTS = wave 2 (v1.37.1): server-side synthesis → cached audio → the EXISTING background-audio machinery; RAM-light (server: ~12GB total/~7.5GB avail, shared); "Listen from Here" required; audio lives in the book experience only (architected for later audiobook export).

## Data model — NEW `db.books` namespace (ytdlp-style). NOT db.metadata.

Rationale: the Phase-2 scan merge (server.js:2898-3149, `mergeScannedMetadata` at 944/3139) wholesale-replaces `fresh.metadata` from a stale Phase-1 snapshot — the 5-strike persist-gate class. A namespace no media-scan writer touches is structurally protected by `updateDatabase`'s untouched-keys round-trip (same guarantee `db.ytdlp.pins` relies on, lib/ytdlp/store.js:1707-1726).

```js
db.books = {
  folders: [],   // operator book roots (must NOT overlap db.folders — reject on save)
  items: {},     // id (getMediaId(filePath), server.js:993) -> book item
  progress: {},  // id -> { locator, percent, updatedAt }
  pins: [],      // shelves — ports the ytdlp pin reducers
  settings: {},  // RESERVED for wave-2 TTS (engine/voice/rate) — never db.settings (13-key lock intact)
}
// item: { id, filePath, rootFolder, folderName, size, mtimeMs, addedAt,
//   format: 'epub'|'pdf', title, author, hasCover, coverExt,
//   spine: [{idref, href}],          // EPUB reading order — wave-2 chapter addressing
//   pageCount }                       // PDF, client-backfilled
```

## Scan — `lib/books/` module + server wiring

- `lib/books/zip.js`: hand-rolled pure zip central-directory reader (~120 lines; EOCD scan, central entries, local-header skip, method 0 as-is / method 8 `zlib.inflateRawSync`; graceful bail on ZIP64/encrypted; caps: central dir ≤4MiB, entry ≤16MiB inflated). NO vendored server lib (no-new-server-deps posture; client-vendoring precedent is client-only). Tests build fixture zips with a ~40-line test-helper writer (`zlib.deflateRawSync`) — no binary fixtures.
- `lib/books/opf.js`: tolerant scoped scanner for container.xml rootfile + OPF dc:title/dc:creator/spine/cover (EPUB2 `meta name="cover"` + EPUB3 `properties="cover-image"`). Every parse failure degrades to filename-title/no-cover — never a scan abort.
- `scanBooks()`: own `bookScanState` overlap guard + `rescanRequested` coalescing (port the discipline from server.js:2231/2317-2345). Triggers: boot, `POST /api/books/scan`, after config save, piggyback on the periodic media timer. Unchanged file (path+size) → reuse. Final write = ONE `updateDatabase` mutator touching only `fresh.books`; the ONLY carry-forward: preserve `hasCover/coverExt/pageCount` when this pass didn't produce them (mid-scan client cover backfill race) — dedicated regression test.
- Mount-loss guard: pure `selectPrunableBookIds` (absent root prunes nothing under it; honors existing `db.settings.pruneMissing` — NO new settings key).
- Covers: `DATA_DIR/.bookcovers/<id>.jpg|.png` (books-owned, never THUMBNAIL_DIR). EPUB covers extracted at scan. PDF: SVG placeholder + one-shot client backfill — the reader (pdf.js has page 1 anyway) renders page 1 → JPEG ≤512KB → `POST /api/books/:id/cover` (magic-byte sniffed per server.js:3671-3675, no-clobber, atomic tmp+rename; `?pages=` int rides along). Template: `POST /api/videos/:id/dimensions` (server.js:5232).

## Reader

- Vendored CLIENT assets under `public/vendor/` (lazy-loaded by the reader view ONLY): epub.js v0.3.93 (BSD-2) + JSZip 3.10 (MIT) + pdf.js v4 (Apache-2, ESM + worker file same-origin). ~1.75MB total (record actuals in vendor README + LICENSE/VERSION files; source-lock test asserts licenses present).
- Views: `public/books.html`+`books.js` (view 'books', route `/books`), `public/read.html`+`read.js` (view 'read', `/read.html?b=<id>`). Router surgery (deliberate lock updates): `deriveRouteView` (common.js:2328), generalize `ensureSubscriptionsScriptLoaded` → `ensureScriptLoaded(src)` (common.js:2574-2592), `activeNavItem` (common.js:1399). "Books" nav link probe-gated on ≥1 configured folder (mirrors injectSubscriptionsNavLinkIfEnabled, common.js:1437) — books-less installs byte-identical.
- Reader chassis: slim top bar (back/title/TOC/Aa), content pane, bottom progress strip; font size + themes persisted in localStorage; `ReaderAdapter` interface (open/display(locator)/currentLocator()/toc()/onRelocated) with epub.js + pdf.js implementations.
- Position schema (ONE shape both formats):
  `locator: {kind:'epub', cfi, spineIndex, blockIndex} | {kind:'pdf', page}` + `percent`.
  CFI = exact restore (reflow-stable). `blockIndex` = wave-2 listen-from-here bridge: index of the CFI's containing element among block-level elements (`p,h1..h6,li,blockquote,pre,figure,td`) in document order — a PURE exported helper in read.js, dual-implementable server-side against the same chapter XHTML via zip.js. PDF listen-from-here = page granularity (accepted).
- epub.js locations: generate(600) once per open, cache JSON in localStorage keyed by id+size.

## Routes (server.js `// ---- Books ----`, delegating to lib/books/*)

- Shelf pins: `GET/POST/DELETE /api/books/pins(/:id)`, `POST /api/books/pins/reorder`.

## Wave-2 (v1.37.1) seams — design-only, reserved now

- `db.books.settings` (ttsEnabled/engine/voice/rate) + `db.books.audio[bookId][spineIndex] = {status,key,durationSec}` (audioStatus-style no-clobber writes, evicted like clearAudioStatus).
- Cache: `DATA_DIR/tts-cache/<key>.m4a` + `<key>.blocks.json` (`[{blockIndex,startSec}]` — THE listen-from-here index: ensure chapter → seek blocks[blockIndex].startSec; no per-offset entries). Key = sha1(bookId:spineIndex:engine:voice:rate:ttsRev).
- Queue: single-worker serialized FIFO (runExclusive-style — long, cancelable jobs, gate introspection). RAM-light BY ARCHITECTURE: one engine process at a time, spawned per job, exits after (zero resident between chapters), synthesizing ahead of playback. Engines emit WAV; EXISTING ffmpeg encodes to m4a (arg-array spawn, no shell).
- Playback: `GET /book/:id/tts/:spineIndex` sendFile → the existing hidden-audio + MediaSession machinery (book cover art via the resolveAudioArtUrl precedent).
- Engine: default **Piper low/medium voice** (models 22-75MB; ~150-300MB transient peak; RTF<1 even on a Pi 5 — owner's 12GB box is comfortable; his prior "lag" burn is neutralized by pre-synthesis). Guaranteed fallback: **espeak-ng** (~5-20MB, robotic, always works). Rejected: Kokoro (RAM). Strictly opt-in like yt-dlp; absent engine = reader still fully functional.

## Risks

1. Persist-gate: ZERO new db.metadata fields (the point of the namespace). Books-internal residual: mid-scan cover backfill vs scanner final write — scoped 3-field carry-forward + regression test. Any future db.books.items field written outside the book scanner needs the same audit.
2. Wild EPUBs vs hand-rolled zip: fail-to-placeholder, never scan-abort; caps; fixture matrix. Residual risk cosmetic.
3. Shared-renderer href widening touches ytdlp render path — bounded by updated shape-lock + 4-combination enablement coverage.
4. pdf.js ESM worker pathing on iOS PWA: needs Dean's device pass (no automated coverage).
5. db.json growth: tens of KB at hundreds of books — negligible.
6. Router/nav lock churn: deliberate-update items, flagged for review.
