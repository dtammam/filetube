# v1.45.6 — Library view preferences (per-page sort + card/list view)

**Status:** in implementation. **Gate:** slim adversarial (no data-loss surface).
Dean's on-device pass is the arbiter. Two small display-preference features for
the library toolbar, requested together.

## #1 — Per-page sort (behind a client toggle)

**Problem:** sort persists GLOBALLY today (`localStorage.filetube_sort`, one key
for every page). Dean wants each folder/page to remember its own sort, behind a
setting flag (default off — preserves today's single-sort behavior).

**Acceptance (observables):**
- Setting **off** (default): one sort applies everywhere — today's behavior,
  byte-unchanged.
- Setting **on**: sort folder A "newest", folder B "title A–Z"; leave and return
  to A → still "newest". Each page independently remembered, across reload.
- Keyed per page = the `?root=<folder>` the page shows (base home = a fixed key;
  `?liked=1` its own). Per-device (localStorage), like the sort it governs.

**Approach:**
- `perPageSort` is a CLIENT localStorage flag (`ft-per-page-sort`), toggled from
  a Settings checkbox — same pattern as `ft-debug-lifecycle` (synchronous at
  read time, no settings-fetch timing games; it governs a per-device pref
  anyway). NOT a server db.settings value.
- Pure helpers (common.js): `pageSortKey(params)` → the page's key;
  `isPerPageSortEnabled()`/`setPerPageSortEnabled()`; the per-page store is a
  JSON map `ft-sort-by-page` = `{ "<key>": "<sort>" }`.
- main.js: at sort-read time, if enabled, resolve `map[key]` (falling back to
  the global `filetube_sort` / defaultSort); on a sort pick, write `map[key]`
  when enabled, else the global key (today's path). Global path untouched when
  off.

## #2 — Card / Compact-list view toggle

**Problem:** only a card grid today; Dean wants a denser list view to see more
per screen, chosen from the filter bar.

**Acceptance (observables):**
- A view-toggle button in the filter bar; **Card** default. Switch to
  **Compact list** → the same items reflow to denser rows (small thumbnail left,
  title+meta right, many per screen). Choice persists across nav/reload
  (per-device, `ft-view-mode` = card|list). The bar stays one clean line
  (tighten sort/shuffle/rescan spacing to fit the extra glyph).

**Approach (CSS-only reflow, no markup fork):**
- `.video-grid.list-view` → single column; `.list-view .video-card` → horizontal
  row (card-media small/left, video-info right), compact. Same card markup.
- New SVG-mask glyphs `icon-grid` / `icon-list` (grid_view / view_list,
  Material). One toggle button `#view-mode-btn` in `.section-actions` showing the
  TARGET view's glyph + a title/aria-label ("Switch to list/card view").
- Pure helpers (common.js): `getStoredViewMode()`/`setStoredViewMode()` (default
  'card'), mirroring the format-toggle persistence. main.js applies the class to
  `#video-grid` on init/render and wires the toggle.
