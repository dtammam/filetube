## 1. Goal (Dean's words)

A proper, first-class **Music library** — a music-app experience, NOT bundled
into the ytdlp module. Its own menu entry, search by song/artist/album, sort,
album art, and the same experience on desktop and iOS PWA including playback
position. Books is the architectural precedent (own nav entry, own
feature-owned namespace, own scan-folder config, own `lib/` module) — copy the
slice pattern, not the internals.

## 3. Architecture directives (hard-won repo lessons — violate NONE)

- **Feature-owned namespace**: new SQLite tables through the `lib/db/sqlite.js`
  adapter seam ONLY (a source-lock test forbids `require('node:sqlite')`
  elsewhere). Own the namespace like Books did → avoids the persist-gate class.
- **A NEW ID-KEYED NAMESPACE IS A DATA-LOSS SURFACE (7th+ strike class)**:
  every id-keyed carrier — delete, prune, move/re-key, backup/restore — must
  carry the music rows in the SAME commit that creates them. The backup bundle
  gains the music namespace + per-user music state THIS wave
  (`BACKUP_NAMESPACE_KEYS` + restore validation + tests). Recompute the 32mb
  bundle-cap headroom WITH music metadata and pin it with a near-cap positive
  test (v1.43.1 backup-restore tests are the template).
- **Scan/prune rigor**: any prune of missing files gets the FULL two-reviewer
  gate. Resurrect/identity tests MUST use DIVERGENT on-disk spellings (v1.41.9
  lesson). NEVER prune on a transient stat/readdir error (unreadable-dir
  conservatism).
- **ffprobe**: reuse the stub-ffmpeg test harness (CI has no ffmpeg); check
  `test/unit/ffprobe-tags*.test.js` for the tags-whitelist precedent before
  extending. Album art → a DATA_DIR cache dir (`.albumart`) mirroring
  `.thumbnails`/`.bookcovers` lifecycle incl. prune.
- **Auth**: the v1.43 gate covers new routes automatically (`app.use` before
  routes). Music pages are post-auth — allowlist NOTHING new. New routes land
  in the route-census test deliberately. Per-user state keys by `req.user.id`
  via `lib/auth/store.js`; reuse/extend the progress coalescer contract (one
  txn per flush window, ≥5:1 write-amp; filter deleted users before the txn).
- **Player**: extend player.js's audio mode, do NOT rebuild it (iOS
  background-audio machinery is battle-won). MediaSession gets
  title/artist/album/art. Next/prev/shuffle ride the registered trackNav
  handlers; the v1.40 ctx-param precedent (context-aware next/prev re-fetches
  the same list query, server order VERBATIM). The >10-min smart-resume
  threshold = ONE pure exported helper with unit tests.
- **UI**: view markup INSIDE `#view-root`, styles in style.css only (v1.41.8
  SPA lesson); typography via `--fs-*` tokens only (lock tests forbid literal
  font-size); glyphs in CSS/SVG never emoji codepoints; `[hidden]{display:
  none!important}` on any hide-via-hidden element. Measure layout, don't guess.
- **Express**: static-segment routes before `/:id` params; route-scoped body
  parsers need their own 4-arg error middleware.

## 4. Technical design

Design grounded in three reconnaissance passes (all file:line current as of
2026-07-17). New server modules: `lib/music/{scan,store,tags}.js` (mirroring
`lib/books/{scan,store,opf}.js`). New client: `public/music.html` +
`public/js/music.js`. The Books slice is the structural template throughout.

### 4.1 Data model & persistence namespaces

New `db.music` container (mirrors `db.books`), registered in `lib/db/sqlite.js`:
- Add `'music'` to `CONTAINER_KEYS` ([lib/db/sqlite.js:88](lib/db/sqlite.js#L88)).
- `DOC_KV_NAMESPACES` ([:59-69](lib/db/sqlite.js#L59)) gains **`'music.tracks'`**
  (one row per track id — the id-keyed track record map).
- `SINGLETON_NAMES` ([:72-83](lib/db/sqlite.js#L72)) gains **`'music.folders'`**,
  **`'music.settings'`** (whole-blob config).
- `KNOWN_SUBKEYS` is auto-built from those two lists — no separate edit.
- `importParsedJson` / `exclusiveReplace` also read those two arrays — covered.

In-memory shape via `lib/music/store.js` `ensureMusic(db)` (mirrors
`ensureBooks`, [lib/books/store.js:29-48](lib/books/store.js#L29)):
```js
db.music = { folders: [], tracks: {}, settings: {} };
```
Plus a non-mutating `readMusic(db)` for GET routes (never mutate the cached DB
— the [lib/books/store.js:59-74](lib/books/store.js#L59) `readBooks` invariant).

**Track record** (`db.music.tracks[id]`), id = `getMediaId(filePath)` (md5 of
absolute path, [server.js:1264](server.js#L1264) — same scheme as video/books):
```
{ id, filePath, rootFolder, size, ext,
  title, artist, album, albumArtist, trackNo, discNo, durationSec, year,
  genre, hasArt, codec, needsTranscode,
  // derived grouping keys computed at read time, not stored:
  //   albumKey = `${albumArtist||artist}␟${album}` (sortable, collision-safe)
}
```
Metadata source precedence per Dean's lock: embedded ffprobe tags → path
convention (`.../Artist/Album/NN Title.ext`) fallback for any missing field.

Per-user music state — three relational tables (schema migration §4.14):
- `user_music_liked(user_id, track_id, liked_at)` PK `(user_id, track_id)`.
- `user_music_progress(user_id, track_id, position_seconds, duration_seconds,
  updated_at)` PK `(user_id, track_id)` — per-track resume position (feeds the
  >10-min smart-resume rule).
- `user_music_state(user_id, last_track_id, queue_ctx_json, updated_at)` PK
  `(user_id)` — the single "resume where I left off" pointer (last track + its
  queue context descriptor). Powers Continue-listening + app-relaunch resume.
All three: `user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE`
(the cascade that makes user-delete wipe music state for free —
[lib/db/sqlite.js:317](lib/db/sqlite.js#L317) `PRAGMA foreign_keys=ON`).

### 4.2 Scan + metadata extraction (`lib/music/scan.js` + `lib/music/tags.js`)

Mirror `lib/books/scan.js` exactly:
- `MUSIC_EXTENSIONS = new Set(['.mp3','.m4a','.aac','.flac','.wav'])` (Dean's
  format lock; no ogg/opus). NB the MAIN media scan already accepts these
  ([server.js:774](server.js#L774)) — the **disjoint-folder rule (§4.4) is what
  keeps a file from being double-claimed**; music folders are a separate tree.
- `walkMusicRoot` — iterative stack walk, symlinked dirs not followed,
  **unreadable dir → `console.warn` + `continue`, never throw** (the
  [lib/books/scan.js:49-54](lib/books/scan.js#L49) conservatism).
- Reuse identity: `existing.filePath === filePath && existing.size === stat.size`
  → reuse byte-for-byte (mtime NOT in identity — [scan.js:166](lib/books/scan.js#L166)).
- Transient mid-walk `stat` failure → drop from `survivingIds` this pass only,
  never add to `missingRoots` ([scan.js:157-162](lib/books/scan.js#L157)).
- `missingRoots` = roots failing `fs.existsSync`.
- Cooperative yield every 8 files (`setImmediate`).

**Tag extraction** (`lib/music/tags.js`, a pure parser + server spawn site):
- Extend `EMBEDDED_TAG_WHITELIST` ([server.js:2191-2194](server.js#L2191)) with
  `'track'`, `'albumartist'`, `'disc'` (title/artist/album/date already present).
- `parseFfprobeTags` ([server.js:2195-2219](server.js#L2195)) already lowercases
  and whitelist-filters; extend its post-processing to surface
  trackNo/discNo/albumArtist. Add cases to
  `test/unit/ffprobe-tags-whitelist-gaps.test.js`
  (the designated additive-coverage home) — hand-built JSON, **no ffmpeg in
  CI** ([docs/RELIABILITY.md:29-33]).
- **Path-convention fallback** — a NEW pure helper `parsePathConvention(
  filePath, rootFolder)` → `{artist, album, trackNo, title}` from
  `.../<Artist>/<Album>/<NN> <Title>.ext`, applied field-by-field only where the
  embedded tag is missing. Pure + unit-tested (its own test file).
- **Album art**: embedded `attached_pic` stream (already probed by
  `buildFfprobeArgs` / `parseFfprobeStreams`, [server.js:2560](server.js#L2560))
  requires a SEPARATE ffmpeg extract pass (`ffmpeg -i f -an -map 0:v -c copy`),
  gated on `ffmpegAvailable`; on miss, look for `cover.jpg`/`folder.jpg` in the
  track's directory. First hit wins → written to `.albumart/<albumKeyHash>.jpg`
  (art is per-ALBUM, deduped by albumKey, not per track — avoids N copies of a
  12-track album's identical cover).

### 4.3 Album-art cache dir (`.albumart`) lifecycle + prune

Mirror the book-cover model (NOT the LRU transcode model):
- `const ALBUMART_DIR = path.join(DATA_DIR, '.albumart')` beside
  `BOOKCOVER_DIR` ([server.js:118](server.js#L118)) — module-owned dir so no
  other scan's unlink can touch it.
- Write best-effort tmp+rename BEFORE the DB merge (book precedent
  [server.js:5229-5244](server.js#L5229)).
- After `updateDatabase` commits and yields `prunedIds`, unlink orphaned
  `.albumart/<key>.{jpg,png}` for album keys no surviving track references
  (album-keyed, so only unlink when the LAST track of an album is pruned),
  best-effort, post-commit (mirror [server.js:5330-5334](server.js#L5330)).
- `GET /albumart/:id` serves the real file (`Cache-Control: public,
  max-age=86400`) or an escaped SVG placeholder (mirror `/bookcover/:id`,
  [server.js:5701-5735](server.js#L5701)). `:id` = track id → resolve to its
  albumKey → serve album art (one art per album).
- **No** independent size/age eviction — art lives and dies with its album's
  tracks (the thumbnail/cover rule, never the TRANSCODE_DIR LRU rule).

### 4.8 Smart-resume helper (>10-min threshold, pure + unit-tested)

ONE pure exported helper beside `shouldShowResumeOverlay`
([player.js:384](public/js/player.js#L384)):
```js
// Restart a track from the top UNLESS it is long (mixes/sets resume mid-track).
function shouldResumeMidTrack(ctx) {
  var dur = Number(ctx && ctx.durationSeconds) || 0;
  var threshold = Number(ctx && ctx.thresholdSeconds) || 600; // 10 min
  return dur > threshold;
}
```
Consulted in `handleResumePlayback` ([player.js:2846](public/js/player.js#L2846))
for `type:'audio'` music items: if `shouldResumeMidTrack({durationSeconds:
currentData.duration})` → resume at saved position; else start at 0 (ignore
saved position for the seek, though the "Continue listening" pointer still
remembers the track). Threshold exported + table-driven unit tests
(0/9:59/10:00/10:01/60min, missing/NaN duration). App-relaunch "Continue
listening" resumes the `user_music_state` track and applies the SAME helper.

### 4.9 On-demand transcode for ALAC/incompatible formats

FLAC/WAV/MP3/AAC/M4A stream natively (verify on-device). Only ALAC (codec
`alac`, usually in `.m4a`) needs transcoding. `needsTranscode`/
`reconcileTranscode` currently HARD-SKIP audio items ([server.js:3596](server.js#L3596),
[server.js:2154-2158](server.js#L2154)) — but those govern the VIDEO
`db.metadata` path, which music does not use. For music we own the decision:
- At scan, ffprobe reports the audio codec; `lib/music/tags.js` sets
  `track.codec` + `track.needsTranscode = (codec === 'alac')` (a small
  music-owned `PLAYABLE_AUDIO_CODECS`-style set; conservative allowlist).
- `GET /track/:id`: if `needsTranscode` and no cached rendition, enqueue via
  the existing format-agnostic queue (`queueAudioExtract`/
  `processAudioExtractQueue` [server.js:2059-2147](server.js#L2059), whose
  `buildAudioExtractArgs` already produces an AAC `.m4a`) targeting a
  `TRANSCODE_DIR/<id>.m4a` rendition; serve `503 {status}` until ready, then
  `sendRangeable` the rendition (the AVI→MP4 precedent, audio flavor).
- **Isolation / risk flag**: ALAC is the highest-risk, most-separable piece
  (Apple-only encoding; plays natively on Apple browsers where Dean mostly
  is). If it destabilizes the wave it can ship as a fast-follow with native
  formats disclosed — decided at the gate, not silently dropped.
